# 项目治理与维护流程 TODO

## 目标

把隐含在代码和维护者经验中的规则固化为可执行、可交接的项目流程。

## TODO

- [ ] 增加根目录 `CONTRIBUTING.md`，说明环境搭建、验证命令和 PR 流程。
- [ ] 增加 `SECURITY.md`、`CHANGELOG.md`、维护者名单和支持版本说明。
- [ ] 建立 `doc/adr/`，记录影响多个 crate 或长期兼容性的架构决策。
- [ ] 为 ADR 定义 proposed、accepted、superseded 状态及评审规则。
- [ ] 定义 Issue/PR 模板，要求风险、测试、兼容性和文档影响说明。
- [ ] 明确各 crate、协议和持久化格式的 code owner。
- [ ] 制定发布检查表：版本、兼容测试、安全扫描、性能、迁移和回滚。
- [ ] 建立技术债务台账，每项包含影响、负责人、优先级和截止条件。
- [ ] 每季度清理过期 feature、兼容层、豁免、TODO 和废弃 API。
- [ ] 在 README 中链接架构、贡献、发布、安全和维护文档。

## 验收标准

- [ ] 新贡献者可以只依赖仓库文档完成环境搭建和首个 PR。
- [ ] 重大设计决策、责任人和发布流程均可追溯。
- [ ] 技术债务和临时豁免不会无限期存在。
